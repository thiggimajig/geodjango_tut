404 Replace values given in `to_replace` with `value`. 

405 

406 Values of the {klass} are replaced with other values dynamically. 

407 {replace_iloc} 

408 

409 Parameters 

410 ---------- 

411 to_replace : str, regex, list, dict, Series, int, float, or None 

412 How to find the values that will be replaced. 

413 

414 * numeric, str or regex: 

415 

416 - numeric: numeric values equal to `to_replace` will be 

417 replaced with `value` 

418 - str: string exactly matching `to_replace` will be replaced 

419 with `value` 

420 - regex: regexs matching `to_replace` will be replaced with 

421 `value` 

422 

423 * list of str, regex, or numeric: 

424 

425 - First, if `to_replace` and `value` are both lists, they 

426 **must** be the same length. 

427 - Second, if ``regex=True`` then all of the strings in **both** 

428 lists will be interpreted as regexs otherwise they will match 

429 directly. This doesn't matter much for `value` since there 

430 are only a few possible substitution regexes you can use. 

431 - str, regex and numeric rules apply as above. 

432 

433 * dict: 

434 

435 - Dicts can be used to specify different replacement values 

436 for different existing values. For example, 

437 ``{{'a': 'b', 'y': 'z'}}`` replaces the value 'a' with 'b' and 

438 'y' with 'z'. To use a dict in this way the `value` 

439 parameter should be `None`. 

440 - For a DataFrame a dict can specify that different values 

441 should be replaced in different columns. For example, 

442 ``{{'a': 1, 'b': 'z'}}`` looks for the value 1 in column 'a' 

443 and the value 'z' in column 'b' and replaces these values 

444 with whatever is specified in `value`. The `value` parameter 

445 should not be ``None`` in this case. You can treat this as a 

446 special case of passing two lists except that you are 

447 specifying the column to search in. 

448 - For a DataFrame nested dictionaries, e.g., 

449 ``{{'a': {{'b': np.nan}}}}``, are read as follows: look in column 

450 'a' for the value 'b' and replace it with NaN. The `value` 

451 parameter should be ``None`` to use a nested dict in this 

452 way. You can nest regular expressions as well. Note that 

453 column names (the top-level dictionary keys in a nested 

454 dictionary) **cannot** be regular expressions. 

455 

456 * None: 

457 

458 - This means that the `regex` argument must be a string, 

459 compiled regular expression, or list, dict, ndarray or 

460 Series of such elements. If `value` is also ``None`` then 

461 this **must** be a nested dictionary or Series. 

462 

463 See the examples section for examples of each of these. 

464 value : scalar, dict, list, str, regex, default None 

465 Value to replace any values matching `to_replace` with. 

466 For a DataFrame a dict of values can be used to specify which 

467 value to use for each column (columns not in the dict will not be 

468 filled). Regular expressions, strings and lists or dicts of such 

469 objects are also allowed. 

470 {inplace} 

471 limit : int, default None 

472 Maximum size gap to forward or backward fill. 

473 regex : bool or same types as `to_replace`, default False 

474 Whether to interpret `to_replace` and/or `value` as regular 

475 expressions. If this is ``True`` then `to_replace` *must* be a 

476 string. Alternatively, this could be a regular expression or a 

477 list, dict, or array of regular expressions in which case 

478 `to_replace` must be ``None``. 

479 method : {{'pad', 'ffill', 'bfill', `None`}} 

480 The method to use when for replacement, when `to_replace` is a 

481 scalar, list or tuple and `value` is ``None``. 

482 

483 .. versionchanged:: 0.23.0 

484 Added to DataFrame. 

485 

486 Returns 

487 ------- 

488 {klass} 

489 Object after replacement. 

490 

491 Raises 

492 ------ 

493 AssertionError 

494 * If `regex` is not a ``bool`` and `to_replace` is not 

495 ``None``. 

496 

497 TypeError 

498 * If `to_replace` is not a scalar, array-like, ``dict``, or ``None`` 

499 * If `to_replace` is a ``dict`` and `value` is not a ``list``, 

500 ``dict``, ``ndarray``, or ``Series`` 

501 * If `to_replace` is ``None`` and `regex` is not compilable 

502 into a regular expression or is a list, dict, ndarray, or 

503 Series. 

504 * When replacing multiple ``bool`` or ``datetime64`` objects and

520 * Regex substitution is performed under the hood with ``re.sub``. The
